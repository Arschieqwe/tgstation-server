1 # API Controllers
2 
3 Most of these controllers map to their specific routes. See the public functions marked with `[HttpXXX]` attributes for where the web meets code.
4 
5 Some notable exceptions:
6 
7 - [ApiController](./ApiController.cs) is the base class of nearly all API related controllers. It does the following:
8  - Contains code to deny the request if the instance is not present when it should be.
9  - Contains the `IDatabaseContext` and `ILogger` properties for child controllers.
10  - Returns 426 Upgrade Required if the API version in the headers are incompatible with the request.
11  - Returns 400 Bad Request if the headers or the PUT/POST'd model is invalid.
12  - Returns 401 If an `IAuthenticationContext` could not be created for a request.
13 - [BridgeController](./BridgeController.cs) is a special controller accessible only from localhost and is used to receive bridge request from DreamDaemon
14 - [HomeController](./HomeController.cs) contains the code to initially log in and generate an API token for a user.
15 - [TgsAuthorizeAttribute](./TgsAuthorizeAttribute.cs) is a special attribute applied to controller methods to define which rights are required to run a verb.
